# 協作流程規則

## 1) 分支規則

- 禁止直接 push 到 `main`。
- 請使用功能分支：
  - `data/*`：資料與標註更新
  - `exp/*`：訓練與實驗更新
  - `docs/*`：文件更新
  - `fix/*`：程式修正

## 2) Pull Request 規則

- 每次變更都要走 PR。
- 一個 PR 只做一件主要事情。
- PR 標題格式建議：
  - `data: ...`
  - `exp: ...`
  - `docs: ...`
  - `fix: ...`

## 3) PR 必填內容

- 改了什麼
- 為什麼要改
- 證據（指標、輸出樣本、CSV/影片路徑）
- 風險與回退方式

## 4) Main 保護設定（GitHub）

`main` 建議至少開啟：
- 合併前必須有 PR
- 至少 1 位審核通過
- 新 commit 推入後自動作廢舊審核
- 限制可直接 push 的人員

## 5) 階段提醒

- 先完成 Phase 1：優先補低 token gloss，並用量化指標驗收。
- Phase 1 達標前，不啟動 Phase 2。
- Phase 2 範圍：只做 `Qwen3.5-9B` 的 gloss 句子化整合。
